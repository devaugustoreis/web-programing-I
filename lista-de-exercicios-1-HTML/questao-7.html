Letra A

Em um formulário, o atributo "action" define para onde (qual rota) os dados do formulário em questão serão enviados. Caso este parâmetro seja omitido, assume como valor padrão a própria rota atual.
Enquanto isso, o atributo "method" define qual método HTTP será utilizado para realizar o envio. Se omitido, possui o valor GET como padrão.

Letra B

GET
O método GET é utilizado para enviar dados através da própria URL na forma de parâmetros. O pareamento dos parâmetros é feito entre os atributos "name" e "value" dos inputs do formulário.
Uma desvantagem deste método é que o comprimento da URL é limitado a 2048 caracteres.
Idealmente esté método não deve ser utilizado para se enviar dados sensíveis (já que ficam visíveis).

POST
o método POST é utilizado para enviar dados através de uma transação (no corpo da requisição HTTP). Assim como no get, as informações também são pareadas pelos atributos "name" e "value" do campo input.
Diferentemente do get, não há limite de tamanho para informações enviadas através do método post.
Embora este método não deixe os dados tão explicitamente vísiveis como no get (em que aparecem na rota), é possível verificar o payload com os dados enviados na aba "network/redes" do navegador, portanto, ainda são necessárias medidas de segurança adicionais para se enviar dados sensíveis.

Letra C

Podemos tornar o preenchimento de um campo do formulário obrigatório ao adicionarmos o atributo "required" no mesmo.

Letra D

A tag label é utilizada para estabelecer-se um vínculo entre um campo do formulário (normalmente um input) e uma "legenda" que o descreve. O paramento entre uma label e um campo é feito, respectivamente, através do atributo "for" e do "id" do campo.
A utilização desta tag é importante, pois oferece as seguintes vantagens:
- Possui grande importância semântica (principalmente considerando-se leitores de tela);
- Permite uma maior acessabilidade a elementos pequenos, que podem ser selecionados ao clicar-se na label correspondente (referenciada através do atributo "for")

Letra E

O atributo "placeholder" é utilizado para mostrar ao usuário um exemplo de como um campo deve ser preenchido.
Por exemplo, você pode deixar o formato de telefone a ser utilizado "(00) 0 0000-0000" e esta informação permanecerá vísivel no campo até que o usuário selecione o campo (em termos de seletores, acionar o "focus"), momento então em que o input se revelará vazio.

Letra F

Abaixo estão listado alguns tipos de inputs que podem ser utilizados:

Type="text"

Input de uma única linha para textos


Type="password"

Input para senhas que, por padrão, esconde (somente na tela) os caracteres digitados


Type="radio"

Input do tipo "radio", que só permite que o usuário selecione uma dentre um grupo de opções. O vínculo entre as opções é feito através do atributo "name".
O dado enviado é o contido no atributo "value" do campo.


Type="checkbox"

Input do tipo "checkbox", que permite que o usuário selecione nenhuma ou várias opções de uma vez.
O dado enviado é o contido no atributo "value" do campo.


Type="email"

Input específico para campos de e-mail. A depender do navegador, pode fornecer validação automática para este propósito (somente no envio).


Type="tel"

Input específico para telefones. Em dispositivos mobile, pode oferecer um teclado especial para telefones.


Type="number"

Input que só aceita valores numéricos.


Type="range"

Input para valores numéricos que contem um slider para ser manipulado. Por padrão, vai de 0 a 100.


Type="color"

Input específico para selecionar uma cor. A depender do navegador, pode fornecer um "colorpicker" ou não.


Type="date"

Input para campos de data. A depender do navegador, pode fornecer um "datepicker" (calendário) ou não.


Type="file"

Input destinado ao envio de arquivos (contendo botão para realizar o upload).


Type="url"

Input para endereços URL. A depender do navegador, pode fornecer validação automática para este propósito (somente no envio).


Type="hidden"

Input que fica invisível ao usuário. (Contudo, pode ser manipulado facilmente através das ferramentas de desenvolvedor).


Type="reset"

Input que cria um botão de reset capaz de restaurar todos os campos do formulário de volta para o valor padrão.


Type="submit"

Input que cria um botão (submit) para enviar todos os dados do formulário.



Abaixo estão listados alguns atributos que podem ser combinados com alguns dos inputs acima:

value - Define um valor padrão para o campo.
checked - Torna o campo pré-selecionado ao carregar a página. (Compatível com radio e checkbox).
disabled - Desabilita o campo.
min - Especifica um valor mínimo para um campo. Pode esperar valores diferentes a depender do input (por exemplo, date VS number/range).
max - Especifica um valor máximo para um campo. Pode esperar valores diferentes a depender do input (por exemplo, date VS number/range).
step - Determina um valor de "salto" para campos em que for pertinente.
minlength - Especifica o mínimo de caracteres para um campo. Compatível com os types text, email, search, password, tel e url.
maxlength - Especifica o máximo de caracteres para um campo. Compatível com os types text, email, search, password, tel e url.
pattern - Especifica um padrão para o campo. Pode ser combinado com expressões regulares.
readonly - Torna o campo somente em leitura (usuário não pode selecionar ou alterar seu valor).
required - Torna o campo obrigatório.
size - Determina o tamanho (largura em caracteres) do campo.

Letra G

Para garantir que os dados em um campo sigam um padrão específico, podemos utilizar o atributo "pattern" em conjunto com expressões regulares (regEx).

Letra H

A única diferença entre as tags "button" e "input type='button'" está nas informações de estilo automaticamente associadas (pelo navegador) a cada um destes elementos, porém essencialmente as duas podem ser utilizadas com as mesmas finalidades.
Por outro lado, um "input type='submit'" diferencia-se das outras duas citadas, pois o "type=submit" torna este elemento como um gatilho capaz de disparar/enviar as informações do formulário ao ser clicado.
Obs.: Uma tag "button" com "type='submit'" também se torna um gatilho que envia o formulário.

Letra I

Podemos utilizar a tag "select" para criar um menu suspenso em um formulário. Dentro da tag "select", precisamos adicionar uma tag "option" para cada uma das opções. Importante frisar que os dados enviados no formulário não são o texto das "options", mas o valor contido no atributo "value".
